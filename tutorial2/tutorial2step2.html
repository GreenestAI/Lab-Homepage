Step 2: LAMMPS input script

Create a new text file and name it input.lammps. Copy the following lines: 
# LAMMPS input script

variable	T equal 300

units		real
atom_style	full
boundary	p p p
pair_style	lj/cut 14

bond_style	harmonic
angle_style	harmonic
dihedral_style	opls
improper_style	harmonic

special_bonds	lj 0.0 0.0 0.5

read_data	data.lammps
There are a few differences with tutorial 1. First, the pair style is lj/cut, which means the atoms closer that 14 Angstroms from each others interact through a Lennard-Jones potential. There is no coulombic interaction. The bond, angle, dihedral, and improper style specify the potential. The three numbers of the special bonds command are weighting factors for the Lennard-Jones interaction between atoms connected by bond (respectively directly bounded, separated by two bonds, etc.). Then the read data command open the data.lammps file, which contains the information about the box size, atoms positions, etc. Then we are going to create a force field. Create a new text file and name it PARM.lammps. Copy the following lines: 
pair_coeff 	1 1 0.066047 3.4
bond_coeff 	1 469 1.4
angle_coeff 	1 63 120
dihedral_coeff	1 0 7.25 0 0
improper_coeff	1 5 180
The pair_coeff indicates the Lennard-jones parameter for the only type of atom of the simulation, carbon. The bond_coeff (respectively angle_coeff, dihedral_coeff, improper_coeff) indicates the energy and equilibrium distance of the only type of bond (respectively angle, dihedral, improper) in the simulation. The file PARM.file can be included in the simulation by adding the following line to input.lammps: 
include	PARM.lammps
In order to impose a deformation to the sheet, let us isolate the atoms at the bottom edge, the atoms at the top edge, and the other atoms (in the middle) and put them in separate groups: 
group		gcar type 1
variable	xmax equal bound(gcar,xmax)-0.5
variable	xmin equal bound(gcar,xmin)+0.5
region		rtop block ${xmax} INF INF INF INF INF
region		rbot block INF ${xmin} INF INF INF INF
region		rmid block ${xmin} ${xmax} INF INF INF INF
group		gtop region rtop
group		gbot region rbot
group		gmid region rmid
The gcar group contains all the atoms. The three regions correspond respectively to the edges and the middle, thanks to the bound variables that locate the extreme most coordinates along x. Then groups corresponding to atoms located in each regions respectively are defined. We now choose the thermalisation of the system and the dynamics. Add the following lines to input.lammps. 
velocity	gmid create ${T} 48455 mom yes rot yes
fix		mynve all nve
compute	Tmid gmid temp
fix		myber gmid temp/berendsen ${T} ${T} 100
fix_modify	myber temp Tmid
The velocity create command gives initial velocities to the atoms of the group gmid, assuring an initial temperature of 300K for these atoms. NVE integration is applied to all atoms, and a Berendsen thermostat is applied to the atom of the group gmid only. The reason we are not thermalising the atoms of the edge is because their motion will be restrained by the following commands: 
fix 		mysf1 gtop setforce 0 NULL 0
fix 		mysf2 gbot setforce 0 NULL 0
velocity	gtop set 0 NULL 0
velocity	gbot set 0 NULL 0
The setforce commands cancel the force applied on the atoms of both edges during the whole simulation along x and z, and the velocity commands cancel the initial velocities along x and z. Therefore, the atoms of the edge will remain immobile during the simulation (if no other command is applied to them). Let us extract the distance between the two edges as well as the force applied on the edges (the values are extracted before being cancelled by the setforce command). Let us also create a dump file. 
variable	L equal xcm(gtop,x)-xcm(gbot,x)
fix 		at1 all ave/time 10 100 1000 v_L file length.dat
fix 		at2 all ave/time 10 100 1000 f_mysf1[1] f_mysf2[1] file force.dat
dump 		mydmp all atom 1000 dump.lammpstrj
Let us run a small equilibration step. 
thermo		100
thermo_modify 	temp Tmid
timestep	1.0
run		5000
Then, we perform a loop. At each step of the loop, the edge are slightly displaced, and the simulation runs for a short time. 
variable 	var loop 10
label 		loop
displace_atoms	gtop move 0.1 0 0
displace_atoms	gbot move -0.1 0 0
run		1000
next		var
jump 		input.lammps loop
