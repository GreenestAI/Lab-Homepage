Step-by-step tutorial — molecular dynamics for absolute beginners

In order to follow this step-by-step tutorial, you need to download LAMMPS, which is an open source classical molecular dynamics code. You also need a text editor, a molecular visualisation program, such as VMD, and a data plotting program, such as Xmgrace. Finally, I use Python to generate the initial configuration.


Step 1 — data file generation

Before running the simulation, we need to generate a file containing the initial positions of atoms. For the force field we use here (TIP4P/2005), we also need to which atoms are bonded together (O-H bond), and which atoms are constrained by an angle (H-O-H). I am using Python to generate this initial configuration file, but you can use another programming language and get the exact same result.

First we open an empty text file, name it "systemgeneration.py", and import mathematical functions.

from math import *

Then we define the water molecule parameters,

typat=2; # 2 types of atom in total (hydrogen and oxygen)
typbo=1; # 1 type of bond (O-H)
typan=1; # 1 type of angle (H-O-H)
mH=1.00794; # hydrogen mass in grams per mole
mO=15.9994; # oxygen mass in grams per mole
alpha=104.5*pi/180; # O-H-O angle (TIP4P/2005) in radian
r=0.9584; # O-H bond distance (TIP4P/2005) in angstrom
qO=-1.1128; # oxygen charge (TIP4P/2005) in multiple of electron charge
qH=0.5564; # hydrogen charge (TIP4P/2005) in multiple of electron charge
dOO=3.1589; # typical O-O distance in angstrom

the box initial dimensions,

txlo=-15; txhi=15;
tylo=-15; tyhi=15;
tzlo=-15; tzhi=15;

and initiate a few counters.

cptAtom=0; # for atom counting
cptBond=0; # for bond counting
cptAngle=0; # for angle counting
cptMolecule=0; # for molecule counting

Then, I open a temporary file in which I write informations needed by LAMMPS, such as box dimensions, atoms mass,

fichier = open("temp", "w")
fichier.write(str(txlo) + " " + str(txhi) + " xlo xhi \n");
fichier.write(str(tylo) + " " + str(tyhi) + " ylo yhi \n");
fichier.write(str(tzlo) + " " + str(tzhi) + " zlo zhi \n");
fichier.write("\n");
fichier.write("Masses \n \n")
fichier.write("1 " + str(mH) + "\n");
fichier.write("2 " + str(mO) + "\n");
fichier.write("\n");

atom location (note: be careful with the indentations in python),

fichier.write("Atoms \n \n")
x=txlo+dOO/2;
y=tylo+dOO/2;
z=tzlo+dOO/2;
while x <= txhi-dOO/2:
    while y <= tyhi-dOO/2:
        while z <= tzhi-dOO/2:
            cptMolecule+=1;
            fichier.write(str(cptAtom+1)+" "+str(cptMolecule)+" 2 "+str(qO)+" "+str(x)+" "+str(y)+" "+str(z) + "\n");cptAtom+=1;
            fichier.write(str(cptAtom+1)+" "+str(cptMolecule)+" 1 "+str(qH)+" "+str(r+x)+" "+str(y)+" "+str(z) + "\n");cptAtom+=1;
            fichier.write(str(cptAtom+1)+" "+str(cptMolecule)+" 1 "+str(qH)+" "+str(r*sin(alpha)+x)+ " "+str(r*sin(alpha)+y)+" "+ str(z)+"\n");cptAtom+=1;
            z=z+dOO;
        y=y+dOO;
        z=tzlo+dOO/2;
    x=x+dOO;
    y=tylo+dOO/2;
fichier.write("\n");

bond informations,

fichier.write("Bonds \n \n")
while cptBond < 2*cptAtom/3:
    fichier.write(str(cptBond+1)+" 1 "+str(cptBond/2*3+1)+" "+str(cptBond/2*3+2)+"\n");
    fichier.write(str(cptBond+2)+" 1 "+str(cptBond/2*3+1)+" "+str(cptBond/2*3+3)+"\n");
    cptBond+=2;
fichier.write("\n");

and finally angle informations.

fichier.write("Angles \n \n")
while cptAngle < cptAtom/3:
    fichier.write(str(cptAngle+1)+" 1 "+str(cptAngle*3+2)+" "+str(cptAngle*3+1)+" "+str(cptAngle*3+3)+"\n");
    cptAngle+=1;

Finally, I generate the data file from temporary file.

fichier.close()
source = open("temp", "r")
txt = source.read()
source.close()
import os
os.remove("temp")
fichier = open("data.lammps", "w")
fichier.write("# Data file LAMMPS"+ "\n"+ "\n")
fichier.write(str(cptAtom)+" "+"atoms"+"\n");
fichier.write(str(cptBond)+" "+"bonds"+"\n");
fichier.write(str(cptAngle)+" "+"angles"+"\n");
fichier.write("\n");
fichier.write(str(typat)+" "+"atom types"+"\n");
fichier.write(str(typbo)+" "+"bond types"+"\n");
fichier.write(str(typan)+" "+"angle types"+"\n");
fichier.write("\n");
fichier.write(str(txt));
fichier.close()

You can download directly both python file (systemgeneration.py) and generated data file (data.lammps).

Step 2 — reach the equilibrium

Now that we have generated a file with the initial positions of the atoms, we need to equilibrate the water at ambient temperature (300 K) and ambient pressure (1 atm). Let us open another empty document, name it "in0", and save it in the same folder as the "data.lammps" file.

# LAMMPS input file

Let us define variables, for the desired temperature, pressure and timestep. In addition, let us define variables for the measurement of the box volume and water density.

variable T equal 300 # kelvin
variable P equal 1 # atmosphere
variable dt equal 2.0 # fs
variable V equal vol # angstrom3
variable C equal count(all)/vol/3*1e30/6.022e23 # mole per m3

First we need to specify which units we want to use. In our case, "real" is a convenient choice. With this choice of units, the distances are in angstrom in agreement with the data file previously generated, see this page for more details.

units real

We also set all boundaries as periodic,

boundary p p p

and use the "full" atom style, as it allows to use bonds, angles, dihedrals, impropers and charges.

atom_style full

Then, we define the bonds and angles as "harmonic".

bond_style harmonic
angle_style harmonic

Finally, we specify that the interactions between water molecules are a combination of a Lennard-Jones potential, and Coulombic interactions. Note that since we use the TIP4P water model, which is a bit special because the charge of the oxygen is located at a massless point M, which is different than the position of the oxygen atom, we use the following pair style (see the wikipedia page of water models).

pair_style lj/cut/tip4p/long 2 1 1 1 0.1546 10.0

The pair style command indicates that, respectively, the oxygen is the atom type "2", the hydrogen the atom type "1", the bond O-H is the type "1", the angle "H-O-H" is the type "1", distance from O atom to massless charge M is 0.1546 angstrom, and that both LJ and coulombic interactions have a cutoff equal to 10.0 angstroms. Finally, as the long range Coulombic interaction are solved in the reciprocal space, we choose a long range solver.

kspace_style pppm/tip4p 1.0e-4

Now we load the previously generated data file,

read_data data.lammps

define the bond and angle value,

bond_coeff 1 0.0 0.9572
angle_coeff 1 0.0 104.52

give the rules for the neighbor detection,

neighbor 1.0 bin
neigh_modify delay 1

and for the LJ interaction (only oxygen-oxygen are different from 0).

pair_coeff * * 0 0
pair_coeff 2 2 0.1852 3.1589

We give an initial temperature to the water,

velocity all create ${T} 6522333 rot yes dist gaussian

and contrain the water using the shake algorithm.

fix shak all shake 1.0e-4 200 0 b 1 a 1

We maintain the temperature at 300K and the pressure at 1 atm using a fix npt, which regulates the box volume in order to adjust the pressure,

fix mynpt all npt temp ${T} ${T} 100 iso ${P} ${P} 1000.0

and record the evolution of the box volume and water concentration as a function of time.

fix at1 all ave/time 100 1 100 v_V file volume.dat
fix at2 all ave/time 100 1 100 v_C file concentration.dat

Finally we choose the frequency of terminal plotting information and create a dump file in order to, later, plot the molecule trajectories,

thermo 100
dump dmp1 all atom 1000 dump.npt.lammpstrj

and run the simulation for a duration of 10 000 timestep, with a timestep of 2 fs which is conventional for TIP4P/2005.

timestep ${dt}
run 10000

When the simulation is done, we save the final state in order to restart a futur simulation from this configuration.

write_restart restart.npt.equil

Step 3 — water viscosity measurement

The presented measurement is indicative only, and care must be taken, in particular "The computation of diffusion coefficients from molecular dynamics simulation must be done carefully, as the result maydepend on the simulation parameters (cut-off radius or methodto compute electrostatic interactions), but also exhibits asystematic size-dependence arising from the screening of hy-drodynamic flows under periodic boundary conditions" [Tazi2012].

Let us open another empty document, name it "in1", and save it in the same folder as the "restart.npt.equil" file. The first lines are identical to the previous document,

variable T equal 300
variable V equal vol
variable dt equal 2.0
units real
boundary p p p
atom_style full
bond_style harmonic
angle_style harmonic
pair_style lj/cut/tip4p/long 2 1 1 1 0.1546 10.0
kspace_style pppm/tip4p 1.0e-4

but instead of opening the "data.lammps" file, we open the restart file, and reset the simulation time at t=0.

read_restart restart.npt.equil
reset_timestep 0

Then, we define new variables that will be usefull for the calculation of the viscosity. The method is inspired from this page.

variable Nevery equal 10
variable Nrepeat equal 100
variable Nfreq equal ${Nrepeat}*${Nevery}
variable kB equal 1.38065e-23
variable atm2Pa equal 101325.0
variable A2m equal 1.0e-10
variable fs2s equal 1.0e-15
variable unitconvert equal ${atm2Pa}*${atm2Pa}*${fs2s}*${A2m}*${A2m}*${A2m}
variable scale equal ${unitconvert}/(${kB}*$T)*$V*${Nevery}*${dt}
The molecule are thermostated using "nvt" thermostat (canonical ensemble - mechanical system in thermal equilibrium with a heat bath at a fixed temperature) with a reasonable damping (100 fs), controlling the temperature of the molecule at fixed volume.
fix mynvt all nvt ${T} ${T} 100
Note that other choices for the thermalisation would be acceptable, such as for example a berendsten thermostat (which rescale the molecule velocity) combined with a "fix nve". We use "Nfreq" for terminal plotting as it must be the same frequency as the "fix ave/correlate".
thermo ${Nfreq}

In addition, we need to evaluate the xy, xz and yz components of the pressure tensor,

variable pxy equal pxy
variable pxz equal pxz
variable pyz equal pyz

from which we calculate the following correlation function

fix acor all ave/correlate ${Nevery} ${Nrepeat} ${Nfreq} v_pxy v_pxz v_pyz type auto file correlate.dat ave running

and finally extract the value of the viscosity in SI units

variable v11 equal trap(f_acor[3])*${scale}
variable v22 equal trap(f_acor[4])*${scale}
variable v33 equal trap(f_acor[5])*${scale}
thermo_style custom step temp press v_pxy v_pxz v_pyz v_v11 v_v22 v_v33
run 100000
variable eta equal 1000*(v_v11+v_v22+v_v33)/3.0
print "viscosity: ${eta} [mPa.s]"

When running this code, I find a viscosity very close to the 0.855 mPa.s, which is expected for the TIP4P/2005 water model. Other methods can be used to measure the viscosity of a fluid, for example using a shear flow, but the present measurement is elegant because it works at the equilibrium, i.e. in absence of external forcing.
